Mastering Healium

The Ultimate Guide to Crafting Effective Prompts for the Healium Agentic System

1. Introduction to Healium

Healium is an advanced agentic framework designed to analyze codebases, identify issues, automate refactoring, and generate dual-format (Markdown and PDF) technical reports. It operates as a single-prompt system, meaning that each project should be driven by one comprehensive master prompt that contains all necessary context, commands, and artifact specifications.

2. Core Principles of Prompting

The C.C.A. Framework

  • Context: Tell Healium exactly what environment it is operating in. (e.g., "In the Vite/React frontend...")
  • Command: State the exact action required using strong verbs. (e.g., "Analyze", "Refactor", "Generate")
  • Artifact: Specify the desired output format. (e.g., "...and produce a PDF error report with matplotlib charts.")

3. Prompt Archetypes

1. Deep Codebase Analysis Analysis

Use these prompts when you need Healium to audit code, find bugs, or calculate technical debt.

"Analyze the 'src/components' directory. Identify any React state management anti-patterns, calculate the estimated fix times for each issue, and compile the findings into a JSON metadata object."

Why it works: It specifies the target directory, the exact type of issue to look for, and the desired structured output.

2. Refactoring & Implementation Action

Use these when you want Healium to actively change the code.

"Refactor the API calls in 'main.py' to use the local Ollama Llama 3.1 model instead of the Groq API. Ensure fallback logic is in place if the local model is unresponsive."

Why it works: It mentions specific files, specifies the target technology (Ollama), and includes edge-case handling requirements.

3. Dual-Format Reporting Reporting

Use these to trigger Healium's custom reporting pipeline (report_gen.py).

"Generate a comprehensive system health report based on the latest codebase scan. Output a detailed Markdown summary and a professional PDF document. Include matplotlib visualizations showing the distribution of error categories."

Why it works: It explicitly requests both Markdown and PDF formats, and specifies the inclusion of data visualizations.

4. Anti-Patterns (What NOT to do)

  • Too Vague: "Fix my code." (Healium doesn't know where to look or what is broken).
  • Overloaded: "Rewrite the backend, change the database to Postgres, update the CSS, and deploy to AWS." (Break this down into step-by-step tasks).
  • Missing Output Specs: "Find bugs in main.py." (Better: "Find bugs in main.py and output the results as a Markdown table.")

5. Master Prompt (Single Prompt Requirement)

Each Healium project must be driven by a single master prompt that includes all context, commands, and output specifications. Use the template below.

"Healium, perform a full audit of the project repository. Step 1: Scan all files for performance bottlenecks. Step 2: Generate quantitative metadata estimating fix times. Step 3: Run report_gen.py to produce a Markdown summary and a PDF report with visualizations. Execute the steps sequentially without intermediate confirmations."
